**ADR: 2025-11.3 -Выбор технологии программирования клиентского приложения**

**Дата:** 8 ноября 2025 г.

**Статус:** Принято

**Контекст:**  
Нужно выбрать технологию для клиентской части платформы управления рецептами: поддержка веб, Android и iOS, безопасность (OIDC), интеграция с бэкендом (C#/.NET) и возможность переиспользования кода.

**Рассмотренные варианты:**  
- Web: React, Vue.js, Angular, Svelte.  
- Mobile/cross‑platform: React Native, Capacitor/Ionic, Flutter, нативная (Kotlin/Swift).

**Решение:**  
Использовать **React** для веб‑клиента и **React Native** для мобильных приложений (Android/iOS). Организовать монорепозиторий и общую библиотеку компонентов/логики.

**Обоснование:**  
- Большая экосистема и поддержка TypeScript; высокая скорость разработки.  
- Переиспользование бизнес‑логики и типов между веб и мобильной частью.  
- Expo / RN позволяют ускорить мобильный цикл разработки; Next.js/Vite ускоряют веб.  
- Соответствует требованиям безопасности и интеграции с микросервисным бэкендом (token‑based auth, OpenAPI).

**Недостатки решения:**  
- Потребуется адаптация UI/UX под платформы; возможны нативные модули для специфичных API.  
- Нужна настройка и поддержка monorepo/CI для нескольких платформ.  
- Требуется обеспечить безопасное хранение токенов на мобильных (Keychain/Keystore).

**Последствия:**  
- Стандартизировать стек: React + React Native + TypeScript; подготовить шаблоны, shared‑package и Storybook.  
- Настроить CI/CD для веб и мобильных сборок, процессы публикации в магазины.  
- Провести обучение команды по RN, безопасности мобильных токенов и общим паттернам компонентов.
